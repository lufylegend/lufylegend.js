lufylegend库件的使用流程

■详细说明：

lufylegend库件的使用方法的使用非常简单：

1，从http://lufylegend.com/lufylegend下载库件最新版。

压缩包内包含lufylegend-x.x.x.js和lufylegend-x.x.x.min.js两个完整版本，还有lufylegend-x.x.x.simple.js和lufylegend-x.x.x.simple.min.js两个缩减版本，***.min是去除了回车和空格的压缩版本。

simple缩减版本与完整版本的区别在于，将LBox2d，LQuadTree，LTransitionManager，LoadingSample1，LoadingSample2，LoadingSample3，LoadingSample4，LoadingSample5等几个类与引擎的常用部分分离，缩减了引擎的体积。如果用到了被分离的部分的功能的话，可以进行手动加载。

2，在html中将库件引入。

3，调用init函数，初始化库件。

■举例：

<!DOCTYPE html>
<html>
<head>
<meta charset="UTF-8">
<title>demo</title>
</head>
<body>
<div id="mylegend">loading……</div>
<script type="text/javascript" src="../lufylegend-x.x.x.min.js"></script> 
<script>
init(50,"mylegend",800,480,main);
function main(){
	alert("感谢您使用lufylegend库件");
}
</script>
</body>
</html>

lufylegend引擎的可操作属性

LGlobal.preventDefault：移动开发中，当点击canvas的时候是否禁止滚屏，默认为true(禁止)

LGlobal.stageScale：舞台的缩放模式，包括LStageScaleMode.EXACT_FIT，LStageScaleMode.SHOW_ALL和LStageScaleMode.NO_SCALE三种类型。

LGlobal.backgroundColor：舞台的背景色，默认为透明。

LGlobal.aspectRatio：该属性只适用于移动端，用于限制横屏LANDSCAPE或者竖屏PORTRAIT， 默认值为NONE，该属性的设定必须在init函数被调用之前。

LGlobal.destroy：LGlobal.destroy的值为true的时候，在使用removeChild函数的时候会自动清空对象本身，false的时候，在使用removeChild函数的时候则不会自动清空对象本身，为了更好的内存回收，1.8.3版将LGlobal.destroy的值默认设置为true。

LGlobal.keepClear：是否每贞清除整个画面后再重绘，默认值为true，设置为falsh后可以在一定程度上提高效率。

全屏设置

设置全屏时，首先需要设置舞台的缩放模式LGlobal.stageScale：

LStageScaleMode.EXACT_FIT:[静态] 指定整个应用程序在指定区域中可见，但不尝试保持原始高宽比。

LStageScaleMode.SHOW_ALL:[静态] 指定整个应用程序在指定区域中可见，且不会发生扭曲，同时保持应用程序的原始高宽比。

LStageScaleMode.NO_BORDER:[静态] 指定整个应用程序填满指定区域，不会发生扭曲，但有可能会进行一些裁切，同时保持应用程序的原始高宽比。

LStageScaleMode.NO_SCALE:[静态] 指定应用程序的大小是固定的，因此，即使在更改播放器窗口大小时，它仍然保持不变。

设置完舞台的缩放模式之后，调用LSystem.screen(LStage.FULL_SCREEN);就可以实现全屏。

※1.8.0版本之前，全屏设置只能放到init函数之前，1.8.0之后可以将全屏设置放到程序中的任何一处。

※对于手机网页，需要在head标签里加上下面标签。

<meta name="viewport" content="width=device-width,initial-scale=1.0,minimum-scale=1.0,maximum-scale=1.0,user-scalable=no">

■使用举例：

可以将全屏设置放到程序中的任何一处。

init(50,"mylegend",480,288,main);
function main(){  
	//......
	LGlobal.stageScale = LStageScaleMode.EXACT_FIT;
	LSystem.screen(LStage.FULL_SCREEN);
	//......
}

或者将全屏设置放到一开始init之前。

LGlobal.stageScale = LStageScaleMode.EXACT_FIT;
LSystem.screen(LStage.FULL_SCREEN);
init(50,"mylegend",480,288,main);
function main(){  
	//......
	//......
}

■作用：

	self.graphics.drawRect(1,"#FF0000",[100,0,100,100],true,"#FF0000");

控制器

■详细说明：

简而言之，一个控制器就是一个类文件。控制器必须放到Controllers文件夹内，控制器的名字以Controller结尾。

在控制器中，可以使用 this.model 来调用它的模型，使用 this.view 来调用它的视图。

控制器中的函数：

construct() ： 控制器初始化结束后会直接调用此函数，如果有想要在控制器初期化结束后运行的代码，可以写在construct函数内，控制器的construct函数，在模型的construct函数运行之后运行。

loadMvc(name,callback,lastClass) ： 读取一组MVC，name是控制器的名称中去除Controller的部分；callback是回调函数，当MVC的三个文件读取完之后，会自动调用此函数；lastClass需要设定为当前对象this。

load：

this.load.model(names,callback) ： 读取一个或者多个模型。

this.load.view(names,callback) ： 读取一个或者多个视图。

this.load.library(names,callback) ： 读取一个或者多个外部类库。

this.load.helper(names,callback) ： 读取一个或者多个辅助函数文件。

this.load.image(loadData,callback) ： 读取多张图片。
